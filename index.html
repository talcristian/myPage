Description

The goal was to learn Solidworks and build a CNC router capable of machining plywood and aluminum. Done in collaboration with two engineering students — my role was the electronics box, circuit design, and CNC shield programming.

The structure uses v-slot aluminum extrusions with eccentric nut carriages. Sheet metal forms most carriage structures; complex parts are 3D-printed in PLA. The electronics box houses a 24V power supply for the steppers and a 600W transformer for the spindle. An Arduino running GRBL connects via USB to the G-code sender.

Specifications

  • 600W Spindle (max 15,000 RPM)
  • ER11 Collets with HSS End Mill
  • GRBL firmware on Arduino CNC Shield
  • Working volume: 100 × 100 × 20 cm
  • Control via Universal G-Code Sender
  • Emergency stop button

What I Learned

  • Solidworks (parts & assembly design)
  • Electronics wiring & connector selection
  • Dust-free electronics enclosure integration
  • GRBL firmware configuration
  • V-wheels cause vibrations — use linear rails next time

Description

Inspired by my aerospace studies, I wanted to cut foam airfoils with a hot nichrome wire. Since nothing open-source existed at the time, I designed and built my own CNC foam cutter.

I used an Arduino Mega with a Ramps 1.4 board running Marlin v1.1 firmware, and wrote a MATLAB script that imports airfoil CSV files, scales and rotates the profile, then generates G-code for the motors. The nichrome wire is heated at 36V — careful balance between heat and feed rate is required for clean cuts.

Specifications

  • Working volume: 30 × 20 × 150 cm
  • Ramps 1.4 + Marlin v1.1 firmware
  • 36V adjustable wire heating
  • LCD screen for status display

What I Learned

  • Mechanical design for CNC machines
  • Ramps 1.4 control board setup
  • Marlin firmware configuration
  • MATLAB G-code generation from CSV airfoil data

Description

To learn Python and the Raspberry Pi, I built a networked weather display. A Raspberry Pi Zero W runs a MQTT broker, pulls data from the OpenWeather API, and updates an e-paper display every 15 minutes. Multiple ESP8266 microcontrollers in different rooms act as MQTT clients, each sending temperature and humidity data from a BM280 sensor.

The Raspberry Pi Zero W uses Python multithreading to concurrently manage the MQTT server, data collection, and display updates.

Specifications

  • Raspberry Pi Zero W as MQTT host
  • ESP8266 microcontrollers as MQTT clients
  • BM280 sensors via I2C
  • OpenWeather API for regional weather data
  • 15-minute display refresh cycle

What I Learned

  • Python multithreading
  • REST API integration
  • MQTT protocol & IoT networking
  • ESP8266 ecosystem (C++)

Description

I redesigned and improved an open-source pen plotter CNC found online. The original project included no electronics, so I selected a GRBL shield for Arduino Uno as the controller. The Core-XY motion system uses Nema 17 steppers for X and Y, and a servo for pen lift (Z).

Images are vectorized in Inkscape and converted to G-code using the GCodeTools extension, then sent to the machine.

  • GRBL shield with Arduino Uno
  • Core-XY kinematics
  • Image vectorization with Inkscape + GCodeTools

What I Learned

  • Motor control fundamentals
  • Image vectorization workflow
  • CNC G-code programming
